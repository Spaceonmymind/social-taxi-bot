```
social-taxi-bot/
├── bot.js            # Чат-бот: меню, уведомления, панель оператора
├── server.js         # HTTP-сервер: принимает заявки из мини-приложения

    └── index.html    # Мини-приложение MAX (форма заказа такси)
```

## Как работает система

Система состоит из **трёх частей**, которые работают вместе:

```
[Пользователь в MAX]
        │
        ▼
[index.html — форма заявки]  ──POST──▶  [server.js — HTTP-сервер]
                                                  │
                                                  ▼
                                         [bot.js — логика]
                                         ├── Сохраняет заявку в requests.json
                                         ├── Уведомляет пользователя в MAX
                                         └── Уведомляет всех операторов в MAX
```

### Флоу заявки шаг за шагом

```
1. Пользователь открывает мини-приложение через кнопку в боте
2. Заполняет форму: ФИО, телефон, адрес, категория, учреждение, дата/время
3. Нажимает «Отправить заявку»
4. server.js валидирует данные и сохраняет через bot.js
5. Пользователь получает сообщение: «Заявка отправлена, ожидайте подтверждения»
6. Все операторы получают уведомление с данными заявки и кнопками действий
7. Оператор назначает автомобиль → указывает стоимость → подтверждает заявку
8. Пользователь получает финальное сообщение: номер авто, время, стоимость
```

## Что нужно для запуска

### Обязательно

| Что | Где взять |
|-----|-----------|
| **Node.js версии 18 или выше** | [nodejs.org](https://nodejs.org) — скачать LTS-версию |
| **Токен бота MAX** | В мессенджере MAX → найти `@MaxDevBot` → `/start` → «Создать бота» |
| **Числовые ID операторов** | Каждый оператор пишет боту `@getmyid_bot` в MAX — он ответит числом |
| **Сервер с Node.js** | VPS, хостинг или локальная машина для запуска `bot.js` и `server.js` |
| **Хостинг для index.html** | Любой HTTPS-хостинг (см. варианты ниже) |

### Варианты хостинга для `index.html`

| Вариант | Сложность | Стоимость | Как |
|---------|-----------|-----------|-----|
| **GitHub Pages** | Просто | Бесплатно | Загрузить файл на GitHub → Settings → Pages |
| **Vercel** | Просто | Бесплатно | Перетащить папку на [vercel.com](https://vercel.com) |
| **Свой сервер** | Средне | По тарифу | Nginx, папка `/var/www/miniapp/` |

> ⚠️ Хостинг **обязательно должен быть HTTPS** — MAX не открывает мини-приложения по HTTP.

### Варианты хостинга для `bot.js` + `server.js`

| Вариант | Сложность | Стоимость |
|---------|-----------|-----------|
| **VPS (Timeweb, Selectel, REG.RU)** | Средне | ~300 руб./мес |
| **Railway.app** | Просто | Бесплатный тариф |
| **Локальный компьютер** | Просто | Бесплатно (только для тестов) |

## Пошаговый запуск

### Шаг 1 — Создайте бота в MAX

1. Откройте мессенджер MAX
2. Найдите бота `@MaxDevBot` и напишите ему `/start`
3. Выберите **«Создать бота»**, придумайте имя и username
4. Скопируйте полученный **токен** — он выглядит как длинная строка символов

### Шаг 2 — Узнайте числовые ID операторов

Каждый человек, который будет получать заявки и управлять ими, должен:
1. Открыть MAX
2. Найти бота `@getmyid_bot` и написать ему любое сообщение
3. Бот ответит числом — это и есть ID (например: `123456789`)

Соберите ID всех операторов через запятую: `123456789,987654321`

### Шаг 3 — Настройте переменные окружения

```bash
# Скопируйте шаблон
cp .env.example .env

# Откройте файл .env любым редактором и заполните все поля
```

Подробное описание каждого поля — в разделе [Переменные окружения](#переменные-окружения).

### Шаг 4 — Опубликуйте мини-приложение (`index.html`)

**Вариант A — GitHub Pages (рекомендуется, бесплатно):**
1. Зарегистрируйтесь на [github.com](https://github.com)
2. Создайте новый репозиторий (например `social-taxi-miniapp`)
3. Загрузите файл `miniapp/index.html`
4. Зайдите в Settings → Pages → выберите ветку `main` → Save
5. Через минуту получите ссылку вида `https://username.github.io/social-taxi-miniapp/index.html`

**Вариант B — Vercel:**
1. Зайдите на [vercel.com](https://vercel.com)
2. Перетащите папку `miniapp/` в браузер
3. Получите ссылку вида `https://social-taxi-miniapp.vercel.app`

Полученную ссылку вставьте в `.env` как `MINI_APP_URL`.

### Шаг 5 — Подключите мини-приложение к боту в MAX

1. Зайдите на [business.max.ru](https://business.max.ru)
2. Найдите вашего бота → **«Чат-бот и мини-приложение»** → **«Настроить»**
3. Вставьте URL из шага 4
4. Выберите вид кнопки → **Сохранить**

### Шаг 6 — Установите зависимости

```bash
npm install
```

Эта команда скачает все необходимые библиотеки в папку `node_modules/`.

### Шаг 7 — Запустите проект

```bash
# Запуск в продакшене (бот + сервер одновременно)
npm start

# Запуск в режиме разработки (с авторестартом при изменении файлов)
npm run dev
```

Если нужно запустить по отдельности:
```bash
node bot.js      # только бот
node server.js   # только сервер
```

После успешного запуска в консоли появится:
```
🤖 Бот запущен
🌐 Сервер запущен на порту 3000
```

## Переменные окружения

Файл `.env` в корне проекта. **Никогда не публикуйте этот файл** — в нём токен бота.

```env
# ──────────────────────────────────────────────
# ОБЯЗАТЕЛЬНЫЕ ПОЛЯ
# ──────────────────────────────────────────────

# Токен бота — получить у @MaxDevBot в MAX
BOT_TOKEN=ваш_токен_бота

# URL опубликованного index.html (обязательно HTTPS)
# Пример: https://username.github.io/social-taxi-miniapp/index.html
MINI_APP_URL=https://ваш-домен/index.html

# Числовые ID операторов через запятую (без пробелов)
# Узнать свой ID: написать @getmyid_bot в MAX
# Пример: 123456789,987654321
ADMIN_IDS=123456789

# Секретный токен для защиты admin-эндпоинтов сервера
# Придумайте любую строку из букв и цифр (минимум 16 символов)
# Пример: Kx9mP2qL8nR5vT3w
ADMIN_TOKEN=придумайте_случайную_строку

# ──────────────────────────────────────────────
# НЕОБЯЗАТЕЛЬНЫЕ ПОЛЯ (есть значения по умолчанию)
# ──────────────────────────────────────────────

# Порт HTTP-сервера (по умолчанию: 3000)
PORT=3000

# Путь к файлу базы заявок (по умолчанию: ./requests.json)
DB_FILE=./requests.json

# Путь к файлу базы отзывов (по умолчанию: ./feedback.json)
FEEDBACK_FILE=./feedback.json
```

## Статусы заявок

| Статус | Значение | Кто устанавливает |
|--------|----------|-------------------|
| `new` 🆕 | Заявка поступила, ожидает обработки | Автоматически при создании |
| `confirmed` ✅ | Оператор подтвердил, авто назначено | Оператор |
| `completed` 🏁 | Поездка завершена | Оператор |
| `cancelled` 🚫 | Отменена пользователем | Пользователь (не позднее 3 часов) |
| `rejected` ❌ | Отклонена оператором | Оператор |

## Работа оператора в боте

После `/start` оператор видит панель управления с кнопками:

```
📋 Активные заявки   — все заявки со статусом «новая»
📁 Все заявки        — полный список за всё время
📊 Статистика        — счётчики по статусам, заявки на сегодня
```

### Обработка заявки оператором

При нажатии на заявку доступны кнопки:

| Кнопка | Действие |
|--------|----------|
| 🚗 Указать автомобиль | Оператор вводит номер и марку (например: `М976ММ белая LADA Granta`) |
| 💰 Указать стоимость | Оператор вводит сумму в рублях |
| 💬 Комментарий | Оператор добавляет внутреннюю заметку (пользователь не видит) |
| ✅ Подтвердить | Заявка подтверждена — пользователь получает уведомление с данными авто |
| ❌ Отклонить | Заявка отклонена — пользователь получает уведомление |

### Что получает пользователь после подтверждения

```
✅ Ваша заявка подтверждена!

Поездка: 15 июня 2025 в 10:00
Учреждение: Городская поликлиника №1
Адрес назначения: ул. Советская, 166
🚗 Автомобиль: М976ММ белая LADA Granta
💰 Стоимость: 250 руб.

Не забудьте паспорт при посадке. Оплата наличными водителю.
```

### Автомобили

В парке **2 автомобиля**. Система автоматически проверяет занятость:
- Если на выбранную дату оба авто уже заняты — форма не даст отправить заявку
- Оператор вводит номер авто вручную при подтверждении

Номера автомобилей по умолчанию (можно изменить в `bot.js`):
- `М976ММ белая LADA Granta`
- `О412СС серая LADA Granta`

## Ограничения и правила

Все ограничения проверяются **автоматически** — и в форме (`index.html`), и на сервере (`server.js`):

| Правило | Подробности |
|---------|-------------|
| **Срок подачи заявки** | Не менее чем за **3 суток** до поездки |
| **Дни работы** | Только будние дни (Пн–Пт). Заявки на Сб/Вс не принимаются |
| **Время работы** | Пн–Чт: 08:00–17:00, Пт: 08:00–16:00 |
| **Отмена заявки** | Не позднее чем за **3 часа** до поездки |
| **Адрес назначения** | Только социально значимые объекты из списка (не произвольные адреса) |
| **Оплата** | Наличными водителю. Паспорт обязателен при посадке |

### Тарифы

| Маршрут | Стоимость |
|---------|-----------|
| По г. Кургану | 119–342 руб. (зависит от района) |
| За пределами г. Кургана | +22 руб. за каждый км после выезда за черту города |
| Ожидание 30 мин. (лето) | 148 руб. |
| Ожидание 30 мин. (зима) | 188 руб. |

## Частые ошибки

**«Бот не отвечает»**  
→ Проверьте `BOT_TOKEN` в `.env`. Токен должен быть именно от MAX, не от Telegram.

**«Мини-приложение не открывается в MAX»**  
→ URL в `MINI_APP_URL` должен начинаться с `https://`. HTTP не работает.

**«Оператор не получает уведомления о новых заявках»**  
→ Проверьте `ADMIN_IDS` в `.env`. Должны быть числовые ID, разделённые запятой без пробелов.

**«При отправке заявки ошибка 500»**  
→ Посмотрите вывод консоли сервера (`node server.js`). Скорее всего не запущен `bot.js` — оба файла должны работать одновременно (`npm start`).

**«Заявка создаётся, но пользователь не получает уведомление»**  
→ Мини-приложение открыто не через бот MAX, а напрямую по ссылке — `senderUserId` не передаётся. Пользователь должен открывать форму через кнопку в боте.

**«Ошибка: Cannot use import statement»**  
→ Версия Node.js ниже 18. Проверьте: `node --version`. Обновите до LTS.
